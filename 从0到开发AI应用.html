零基础
AI 应用开发教程

从不会编程开始，按阶段补齐计算机、Python、AI 基础、大模型、RAG、Agent、部署和项目落地能力。

作者：wait

最终目标：能独立做出 AI 聊天机器人、知识库问答、AI 助手、私有部署大模型，并具备接单交付能力。

这份路线不追求“学术全懂”，而是用最短路径建立能做项目的知识骨架。

这份学习方案面向完全零基础的人。重点不是一开始就钻算法公式，而是先建立能动手的能力，再逐步理解背后的 AI 原理。整个路线有一个判断标准：每学一段，都要能做出一个更完整的作品。最后不是“看过很多课”，而是能交付聊天机器人、知识库问答、AI 助手和私有模型部署这样的真实项目。

第一阶段看起来简单，但对零基础非常关键。很多人不是输在 AI，而是输在环境配置、路径、终端、文件结构这些基本概念上。建议用两周时间把开发环境摸熟：知道代码文件在哪里，知道命令在哪运行，知道报错怎么复制给 AI 工具分析。这个阶段不要急着学模型，先把“我能让电脑按我的指令运行”这件事建立起来。

Python 是这条路线的第一条硬技能主线。对 AI 应用开发来说，Python 不需要先刷到算法竞赛水平，但必须熟练处理数据、文件、API 和错误。学习时建议用真实小任务练习，比如批量重命名文件、读取 Excel、调用一个天气 API、把网页内容保存成 JSON。只看视频不会形成编程能力，必须每天写代码，并学会让 AI 帮你解释报错。

阶段 3：AI 基础理论，学“够用的原理”

05

推荐学习内容

  • 什么是数据集、特征、标签、训练、推理
  • 分类、回归、聚类的基本区别
  • 准确率、召回率、F1、混淆矩阵
  • 神经网络、梯度下降、过拟合
  • PyTorch 基础：张量、模型、训练循环

学习原则

不要把目标设成“成为算法研究员”。你的目标是能判断模型为什么会错、如何评估、如何把 AI 能力接进应用。

  • 公式看懂含义即可，先不追求推导全会
  • 每个概念配一个小实验
  • 重点理解评估，不要只理解训练
AI 基础理论阶段的定位要讲清楚：不是把普通人培养成论文研究员，而是让他具备判断 AI 系统的基本能力。比如为什么训练集效果好，上线效果差；为什么准确率高但用户觉得不好用；为什么数据质量比模型名字更重要。这些问题都需要一点机器学习基础。建议用 4 到 6 周学习，边学边做小实验，比如手写数字识别、文本分类、简单推荐。

主流大模型速览：先知道有哪些“发动机”

06

闭源商用模型 API 优先

OpenAI GPT-5.5 / GPT-5 系列综合能力强，适合复杂推理、代码、Agent 工作流和正式产品 API 接入。
Claude Opus / Sonnet 系列长文本、写作、代码理解和工具使用体验稳定，适合知识工作与复杂文档任务。
Google Gemini 3 系列多模态、长上下文和 Google 生态集成突出，适合图文音视频理解类应用。

开源 / 开放权重模型 部署优先

Qwen 系列中文、代码和工具调用生态活跃，适合本地实验、企业私有化和中文应用。
Llama / Mistral 系列开放生态成熟，社区工具多，适合学习推理部署、微调和应用集成。
DeepSeek 系列推理和代码方向影响力强，适合学习低成本推理、私有部署和工程化优化。
零基础建议：先用闭源 API 做产品原型，快速理解 AI 应用链路；再用开放权重模型学习本地部署、成本控制、私有化和微调。
这一页帮助学习者建立模型版图。闭源模型适合先做产品，因为 API 稳定、能力强、接入快，能让新人先把聊天、知识库和工具调用这些应用链路跑通。开放权重模型适合后半段学习部署和私有化，因为你能控制模型、硬件、成本和数据边界。要强调：模型列表会变化，不要把学习路线绑定在某一个模型名字上。真正长期有价值的是理解 API、上下文、检索、评测和部署这些通用能力。

阶段 4：大模型基础，进入 LLM 应用开发

07

Prompt

角色、任务、输入边界、输出格式、少样本示例、拒答条件。

API

聊天补全、流式输出、函数调用、错误重试、成本统计。

上下文

Token、上下文窗口、截断、摘要记忆、长文档处理。

可靠性

幻觉、格式失败、评测集、人工复核、安全提示注入。

必做小项目

做一个命令行 AI 助手：输入问题 → 调用大模型 API → 流式返回 → 保存历史 → 处理错误。

大模型阶段开始进入真正的 AI 应用开发。这里重点不是“会问 AI 问题”，而是会把大模型当成一个不稳定但强大的接口来使用。必须学会 API 调用、流式输出、prompt 结构化、JSON 输出、错误重试和成本控制。最好的练习是做一个命令行 AI 助手。它很小，但覆盖了后面所有应用都会用到的核心链路。

阶段 5：微调与部署，理解“什么时候不用微调”

08

必学知识点

  • Embedding 与向量检索
  • LoRA / QLoRA 基本概念
  • 量化、显存、吞吐、延迟
  • Ollama、vLLM、Docker
  • 模型服务 API 化

学习顺序

  • 先会本地运行模型
  • 再学量化和推理服务
  • 再理解微调数据格式
  • 最后做一次 LoRA 实验

关键避坑

  • 不要把所有问题都用微调解决
  • 知识更新优先考虑 RAG
  • 风格和格式问题再考虑微调
  • 私有部署要算硬件成本
这一阶段很容易走弯路。很多新人一听大模型就想微调，但真实项目里，知识库问答通常优先用 RAG，业务流程优先用工具调用，只有稳定风格、特定格式、领域表达等问题才考虑微调。部署也一样，不是能跑起来就结束，要理解显存、并发、延迟、量化和监控。建议先用 Ollama 跑本地模型，再用 Docker 或 vLLM 做一个简单推理服务。

接单与落地：把作品变成可交付方案

15

作品集

  • 每个项目有在线演示
  • 有 GitHub 仓库和 README
  • 有 2 分钟演示视频
  • 写清业务价值和技术栈

接单方向

  • 企业知识库问答
  • 客服/销售 AI 助手
  • 文档自动化处理
  • 内部流程 Copilot

交付清单

  • 需求确认表
  • 原型和报价
  • 部署文档
  • 账号权限和维护计划

报价思路

先从小项目开始：知识库问答 Demo、内部资料助手、自动生成报表。不要承诺“完全替代人”，要承诺明确流程、明确指标、明确维护范围。

最后要把学习转成真实价值。接单不是只会写代码，还要会确认需求、控制范围、交付文档和维护。新人建议先从小项目做起，比如企业内部资料问答、文档摘要、销售话术生成、周报生成。报价时不要承诺玄乎的结果，比如“准确率百分百”或者“完全替代员工”。要把交付范围写清楚：支持哪些文件、多少用户、如何部署、后续维护怎么算。

The One-Line Path

长期照着走的极简主线

先学会写 Python 脚本，再会调大模型 API，再做前后端聊天应用，再接入 RAG 知识库，再加入 Agent 工具调用，最后学会 Docker/私有部署和项目交付。

Python → API → Web 应用 → RAG → Agent → 部署 → 接单

每一段只问一句：我能不能做出一个可演示、可部署、可维护的小作品？

收尾时把路线压缩成一句话：Python 到 API，到 Web 应用，到 RAG，到 Agent，到部署，到接单。只要不偏离这条主线，就不会走太多弯路。遇到新技术时先问：它能不能帮助我做出更稳定、更有价值的 AI 应用？如果不能，先放一放。学完这条线，AI 聊天机器人、知识库问答、AI 助手、私有部署大模型都会成为可完成的项目，而不是停留在概念里。
